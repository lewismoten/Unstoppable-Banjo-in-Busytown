Web RTC Example: Answer

This example needs a peer to make an offer and another peer to answer the offer.

Ready

Establishing a Peer-to-Peer connection

The remote peer will initiate a connection. This peer will take the remote peers offer and create an answer.

Remote Session Description

The remote peer will create a new session description and send it as an offer. The SDP (Session Description Protocol) also known as a session profile describes the communications of the session. It would specify type of data to exchange such as audio and/or video.

Paste the SDP from the remote peer, or the value in this sites key/value store.

Signaling State: Unknown

Local Session Description

A local session description is prepared and sent in response to the offer as an answer.

Copy the SDP and send it to the remote peer, or the value in this sites key/value store.

ICE Gathering State: Unknown
ICE Connection State: Unknown
